# XOR解密工具

一个基于浏览器的XOR解密工具，支持多种输入格式和密钥类型，完全在本地运行，无需服务器。

## 功能特性

- 多种密钥类型支持：文本、十六进制、二进制、十进制
- 多种输入格式：十六进制、Plain Text、Base64、Binary、文件上传
- 自动检测解密后的文件类型
- 内置预览功能（支持图片和文本）
- 一键保存解密结果
- 纯前端实现，数据不上传服务器

## 使用方法

### 1. 打开工具

直接在浏览器中打开 `index.html` 文件即可使用。

### 2. 设置密钥

- 在"密钥设置"区域输入解密密钥
- 选择密钥类型：
  - **Text (文本)**: 直接输入文本密钥
  - **Hex (十六进制)**: 输入十六进制格式的密钥
  - **Binary (二进制)**: 输入二进制字符串格式的密钥
  - **Decimal (十进制)**: 输入0-255之间的十进制数值（用空格或逗号分隔）

### 3. 输入加密数据

选择以下任一方式输入需要解密的数据：

- **十六进制 (HEX)**: 输入十六进制编码的加密数据
- **Plain Text**: 输入纯文本格式的加密数据
- **Base64**: 输入Base64编码的加密数据
- **Binary**: 输入二进制字符串格式的加密数据
- **文件上传**: 选择加密文件，并指定文件内容格式

### 4. 解密

点击"解密"按钮进行解密操作。

### 5. 查看和保存结果

- 解密成功后，会显示文件大小和类型信息
- 图片和文本文件可直接预览
- 点击"保存文件"按钮下载解密后的文件

## 支持的文件类型

工具会自动检测解密后的文件类型，支持：

- 图片：JPEG、PNG、GIF
- 文档：PDF
- 压缩包：ZIP、GZIP
- 文本：纯文本文件
- 其他：二进制文件

## 技术栈

- 纯 HTML5 + CSS3 + JavaScript
- 无需任何依赖或框架
- 使用浏览器原生 API（TextEncoder、TextDecoder、Blob、File API）

## 示例

### 示例1：解密十六进制数据

```
密钥: G0G0Squ1d3Ncrypt10n
密钥类型: Text (文本)
输入格式: 十六进制 (HEX)
数据: b8e8b8eb53327539...
```

### 示例2：解密文件

```
密钥: 47 6f 47 6f
密钥类型: Decimal (十进制)
输入格式: 文件上传
文件格式: 二进制文件 (直接读取)
```

## 注意事项

- 所有解密操作均在浏览器本地完成，数据不会上传到任何服务器
- 请确保密钥正确，否则解密结果将无法识别
- 对于大文件，解密可能需要一些时间
- 建议使用现代浏览器（Chrome、Firefox、Edge等）以获得最佳体验
